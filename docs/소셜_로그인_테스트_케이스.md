## Android 테스트 케이스

### TC-AND-001: 앱 실행 및 로그인 화면 확인

**목적**: Android에서 로그인 화면이 정상적으로 표시되는지 확인

**전제조건**:
- 앱이 설치되어 있지 않거나 로그아웃 상태

**테스트 단계**:
1. 앱 실행
2. 로그인 화면 진입

**예상 결과**:
- [ ] 앱이 정상적으로 실행됨
- [ ] 로그인 화면이 표시됨
- [ ] **카카오로 계속하기** 버튼 표시
- [ ] **Google로 계속하기** 버튼 표시
- [ ] **Apple로 계속하기** 버튼 **숨김** (표시되지 않음)
- [ ] 이메일 로그인 폼 표시

**우선순위**: 높음

### TC-AND-002: Kakao 로그인 (계정 로그인)

**목적**: Kakao 계정으로 로그인 성공 확인

**전제조건**:
- Kakao 테스트 계정 준비
- 카카오톡 앱 미설치 (계정 로그인 테스트)
- Kakao Key Hash 등록 완료 (`Sa32n6shrINDWbUtanGU55Y1RUo=`)

**테스트 단계**:
1. 로그인 화면에서 **카카오로 계속하기** 버튼 클릭
2. 카카오 계정 로그인 웹뷰 표시 확인
3. 이메일과 비밀번호 입력
4. 로그인 버튼 클릭

**예상 결과**:
- [ ] 카카오 로그인 웹뷰가 정상적으로 열림
- [ ] 로그인 성공 후 메인 페이지로 이동
- [ ] 사용자 정보가 정상적으로 표시됨
- [ ] 로그인 상태 유지됨

**실패 시 확인사항**:
- Kakao Key Hash가 등록되었는지 확인
- 로그에서 "Android keyHash validation failed" 에러 확인
- 패키지명이 `com.wpipainfree.connect`인지 확인

**우선순위**: 높음
